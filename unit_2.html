Lists

Dive into the world of lists in Python!

Creation

Definition: List creation involves defining an ordered, mutable collection of items using square brackets `[]`.

Explanation: Think of a list as a shopping list—you can write it down, add or remove items, and rearrange them as needed. Lists can hold mixed data types and are enclosed in `[]` with items separated by commas.

Syntax: 

my_list = [item1, item2, item3]

Examples

Example 1
[1, 2, 3]
Example 2
[1, 'hello', 3.14]
Example 3
[]
Example 4
[10, 20, 30]

Basic Operations

Definition: Basic operations on lists include accessing elements, modifying them, appending new items, and removing elements.

Explanation: Think of a list as a dynamic to-do list—you can check items (access), update them (modify), add new tasks (append), or cross off completed ones (remove). Use indices for access and methods like `append()` or `remove()` for changes.

Syntax: 

list[index]        # Access
list[index] = value # Modify
list.append(item)  # Append
list.remove(item)  # Remove

Examples

Example 1
banana
Example 2
[10, 25, 30]
Example 3
['apple', 'banana', 'orange']
Example 4
[10, 30]

Advanced Notes (Optional)

- Lists are mutable, unlike strings or tuples.
- Use `len(list)` to get the number of items.
- The `insert(index, item)` method adds an item at a specific position.
- `pop(index)` removes and returns an item at the given index.
- Lists can be nested to create multidimensional structures.

Tuples

Explore the immutable world of tuples in Python!

Creation

Definition: Tuple creation involves defining an ordered, immutable collection of items using parentheses `()` or the `tuple()` function.

Explanation: Think of a tuple as a locked treasure chest—once you fill it with items, you can’t change them. It’s great for fixed data like coordinates and uses `()` with commas.

Syntax: 

my_tuple = (item1, item2, item3)

Examples

Example 1
(1, 2, 3)
Example 2
(1, 'hello', 3.14)
Example 3
()
Example 4
(10, 20, 30)

Basic Operations

Definition: Basic operations on tuples include accessing elements, with immutability preventing modification, appending, or removal.

Explanation: A tuple is like a read-only guest list—you can look up names (access) but can’t add or remove anyone once it’s set. Use indices for access, and note that methods like `count()` or `index()` are available.

Syntax: 

tuple[index]        # Access
# No modify, append, or remove

Examples

Example 1
banana
Example 2
Tuples are immutable!
Example 3
Count of 2: 2
Example 4
Index of banana: 1

Advanced Notes (Optional)

- Tuples are immutable, making them faster than lists for read-only data.
- Use `len(tuple)` to get the number of items.
- A single-item tuple requires a trailing comma, e.g., `(1,)`.
- Tuples can be unpacked into variables (e.g., `x, y = (1, 2)`).
- They are often used as dictionary keys due to immutability.

Sets

Discover the unique world of sets in Python!

Creation

Definition: Set creation involves defining an unordered, mutable collection of unique items using curly braces `{}` or the `set()` function.

Explanation: Think of a set as a unique guest list—no duplicates allowed, and the order doesn’t matter. Use `{}` for sets with items or `set()` to convert other iterables, ensuring uniqueness.

Syntax: 

my_set = {item1, item2, item3}

Examples

Example 1
{1, 2, 3}
Example 2
{1, 2, 'hello'}
Example 3
set()
Example 4
{10, 20, 30}

Basic Operations

Definition: Basic operations on sets include adding items, removing items, and performing set operations like union or intersection.

Explanation: A set is like a club membership list—you can add new members (`add()`), remove existing ones (`remove()`), or find common members with another club (union/intersection). Sets maintain uniqueness automatically.

Syntax: 

set.add(item)      # Add
set.remove(item)   # Remove
set1.union(set2)   # Union

Examples

Example 1
{1, 2, 3}
Example 2
{'apple'}
Example 3
{1, 2, 3}
Example 4
{2, 3}

- Sets are unordered and do not support indexing.
- Use `len(set)` to get the number of unique items.
- The `discard()` method removes an item without raising an error if it’s absent.
- Set operations include `intersection()`, `difference()`, and `symmetric_difference()`.
- Sets are efficient for membership testing with `in`.

Dictionaries

Unlock the power of key-value pairs with dictionaries!

Creation

Definition: Dictionary creation involves defining a collection of key-value pairs using curly braces `{}` or the `dict()` function.

Explanation: Think of a dictionary as a phonebook—each entry has a name (key) and a number (value). Keys must be unique and immutable, and pairs are separated by commas within `{}`.

Syntax: 

my_dict = {key1: value1, key2: value2}

Examples

Example 1
{'name': 'Alice', 'age': 25}
Example 2
{'id': 1, 'title': 'Book', 'price': 19.99}
Example 3
{}
Example 4
{'name': 'Bob', 'age': 30}

Basic Operations

Definition: Basic operations on dictionaries include accessing values, modifying values, adding new key-value pairs, and removing pairs.

Explanation: A dictionary is like a dynamic address book—you can look up a name (access), update a phone number (modify), add a new contact (add), or delete an entry (remove). Use keys for access and methods like `update()` or `pop()` for changes.

Syntax: 

dict[key]         # Access
dict[key] = value # Modify/Add
dict.pop(key)     # Remove

Examples

Example 1
Alice
Example 2
{'name': 'Alice', 'age': 26}
Example 3
{'name': 'Alice', 'city': 'New York'}
Example 4
{'name': 'Alice'}

- Dictionary keys must be immutable (e.g., strings, numbers, tuples).
- Use `len(dict)` to get the number of key-value pairs.
- The `get(key)` method returns a value or a default if the key is absent.
- `update(other_dict)` merges another dictionary into the current one.
- Dictionaries preserve insertion order since Python 3.7.

Object-Oriented Concepts

Introduction to Class and Object in Python!

Introduction to Class and Object

Definition: A class is a blueprint for creating objects, which are instances of the class with their own data and behaviors.

Explanation: Think of a class as a cookie cutter and objects as the cookies it produces. The class defines properties (attributes) and actions (methods), while objects are specific instances with their own values.

Syntax: 

class ClassName:
    def __init__(self, attr1, attr2):
        self.attr1 = attr1
        self.attr2 = attr2
    def method(self):
        # Code
obj = ClassName(val1, val2)

Examples

Example 1
Buddy
Example 2
Whiskers says meow!
Example 3
Name: Alice, Age: 20
Example 4
Car 1: Toyota, Car 2: Honda

File Handling

Master file operations in Python!

Creating Files

Definition: Creating files involves using Python to generate new text files using the `open()` function with mode `'w'` or `'x'`.

Explanation: Think of creating a file as starting a new notebook—use `open()` to create it, write content, and close it to save. Mode `'w'` overwrites, while `'x'` fails if the file exists.

Syntax: 

with open('filename.txt', 'w') as file:
    file.write('content')

Examples

Example 1
File created!
Example 2
File created with multiple lines!
Example 3
File created successfully!
Example 4
File created with greeting!

Reading and Writing to Text Files

Definition: Reading and writing to text files involves using `open()` with modes `'r'` (read), `'w'` (write), or `'a'` (append) to interact with file content.

Explanation: Imagine a notebook—you can read its pages (`read()`), overwrite them (`write()`), or add notes at the end (`append()`). The `with` statement ensures the file closes properly.

Syntax: 

with open('file.txt', 'r') as file:
    content = file.read()
with open('file.txt', 'w') as file:
    file.write('new content')

Examples

Example 1
Hello, Python!
Example 2
File updated!
Example 3
Entry appended!
Example 4
['Hello, Python!']

Working with CSV Files

Definition: Working with CSV files involves reading and writing data in comma-separated values format using the `csv` module.

Explanation: Think of a CSV file as a spreadsheet saved as text—each line is a row, and commas separate columns. The `csv` module simplifies reading rows as lists or writing data with `writer`.

Syntax: 

import csv
with open('file.csv', 'w', newline='') as file:
    writer = csv.writer(file)
    writer.writerow(['name', 'age'])

Examples

Example 1
CSV file created!
Example 2
['Name', 'Age'] ['Alice', '25']
Example 3
CSV file created!
Example 4
Name: Alice, Age: 25

Advanced Notes (Optional)

- Use `with` to automatically close files and avoid resource leaks.
- Mode `'r+'` allows both reading and writing.
- The `csv.DictReader` and `csv.DictWriter` handle CSV data as dictionaries.
- Handle file errors with `try-except` for robustness.
- Binary files use modes like `'wb'` or `'rb'` for non-text data.
